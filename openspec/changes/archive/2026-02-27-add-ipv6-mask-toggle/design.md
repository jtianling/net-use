## Context

当前实现将 IPv6 作为完整地址写入 `DiscoveredAddress::Ipv6Full`, 并在 TUI, 导出, 复制, `--no-tui` 中统一输出。  这导致白名单结果过于离散, 且无法在聚合视图和原始地址视图之间快速切换。  同时现有 `MonitorView` 仅维护单一地址列表, 无法表达“同一监控会话中的 IPv4/IPv6 双视图”。

## Goals / Non-Goals

**Goals:**
- 将 IPv6 的默认输出语义切换为 `/64` 掩码地址, 并保持去重行为稳定。
- 在监控页面增加 `S` 快捷键, 允许 IPv4 和 IPv6 同步在“掩码视图”和“原始地址视图”之间切换。
- 保持 IPv4 `/24` 行为不变, 并保证 `--no-tui` 输出仍然一行一个可用于管道处理。
- 让导出和复制默认输出掩码视图, 以保持白名单场景的一致性。

**Non-Goals:**
- 不引入可配置掩码长度 (如 `/48`, `/56`)。
- 不改变地址过滤规则 (loopback, link-local 仍按原规则忽略)。
- 不新增持久化或跨会话保存显示模式。

## Decisions

### 1. IPv6 canonical 输出改为 `/64`

**选择**: 在地址模型中新增 IPv6 `/64` canonical 表示, 所有“输出通道”默认使用 canonical 值。  例如 `2607:6bc0::10` 输出为 `2607:6bc0::/64`。

**备选方案**:
- 继续使用完整地址作为 canonical, 仅在展示层临时格式化 `/64`。
- 在 collector 阶段直接转字符串并丢弃结构化地址。

**理由**: canonical 在数据层统一为 `/64` 可以保证去重, 输出, 导出的语义一致, 避免各层重复掩码逻辑和行为偏差。

### 2. 原始地址独立维护, 不影响 canonical 去重

**选择**: 监控链路同时维护四类集合: `ipv4_masked_set` (按 `/24` 去重), `ipv4_raw_set` (按完整地址去重), `ipv6_masked_set` (按 `/64` 去重), `ipv6_raw_set` (按完整地址去重)。  collector 仍只对 canonical 新增触发“新地址输出事件”, 但会把新原始地址同步给 TUI 状态用于切换显示。

**备选方案**:
- 仅保留每个 `/64` 的首个原始地址样本。
- 切换到原始视图时动态反推 (不可行, `/64` 无法反推完整地址)。

**理由**: 用户需要在聚合和原始视图之间切换, 这要求完整地址集合在采集期间被持续保存, 且不能被 `/64` 去重覆盖。

### 3. TUI 增加统一显示模式状态机

**选择**: 在 `MonitorView` 增加 `AddressDisplayMode` 状态 (`Masked`, `Raw`), 默认 `Masked`。  `S` 键在两种模式间切换时, IPv4 和 IPv6 两个面板同步切换, 标题和状态栏同时展示当前模式。

**备选方案**:
- 使用“按住键临时显示原始地址”。
- 增加独立页面展示原始地址。

**理由**: 二态切换学习成本低, 对现有键位冲突最小, 且与当前单屏布局兼容。

### 4. 导出与复制始终输出 masked 视图

**选择**: `E` 和 `C` 仍以“可直接用于白名单”的输出为主, 即 IPv6 始终写 `/64` 列表, 不受当前显示模式影响。

**备选方案**:
- 导出/复制跟随当前显示模式。

**理由**: 该工具核心目标是产出白名单条目。  固定输出 masked 可避免用户在 Raw 模式下误导出过细规则。

## Risks / Trade-offs

- [Raw 列表增长更快] → 原始 IPv4/IPv6 条目可能显著多于掩码条目, 增加内存和渲染压力。  缓解: 继续使用去重集合, 并保持列表渲染按当前可见区域。
- [事件模型复杂度上升] → 同时维护 masked/raw 可能让状态同步出错。  缓解: 明确 canonical 与 raw 的职责边界, 并补充单元测试覆盖“同 `/64` 多原始地址”场景。
- [导出语义与屏幕可见内容不一致] → 用户在 Raw 视图下导出会得到 masked。  缓解: 状态栏增加简短提示说明导出策略。

## Migration Plan

1. 调整地址类型与聚合逻辑, 先让 canonical IPv6 变为 `/64` 并通过编译。
2. 扩展监控事件/视图状态, 维护 IPv4/IPv6 的 masked 与 raw 双套数据。
3. 增加 `S` 键处理与界面文案, 完成显示切换。
4. 更新 `--no-tui` , 导出, 复制行为及测试用例。

回滚策略: 若上线后发现兼容性问题, 可先回退为“仅 masked 输出, 无 Raw 切换”, 保留 collector `/64` 逻辑。

## Open Questions

- Raw 视图是否需要显示条目上限或分页提示, 以避免大量地址时影响可读性。
- 后续是否需要引入 `--ipv6-mask-len` 配置, 支持企业网络的不同聚合粒度。
